# Testing Principles

## String Assertions

### What is allowed
- Assertions against **controlled literals** you define: class names from fixtures, exact CLI strings
  owned by the codebase (e.g. `"Export failed:"`), known enum values, values from `st.sampled_from()`
- Comparing two generated things to **each other** — but prefer `is` identity when possible

### What is forbidden
- Asserting a string value that was **generated by Hypothesis** or constructed dynamically at runtime
- Broad `.lower()` substring checks on full CLI output:
  ```python
  # Bad — breaks if help text wording changes
  assert "train" in result.stdout.lower()

  # Good — the exit code is the real contract
  assert result.exit_code == 0
  ```

### Structural assertions for dynamic imports
When testing `import_from_path` / `import_from_module` and similar dynamic import functions:
```python
# Bad — asserts a generated name as a string
assert result.__name__ == generated_name

# Good — structural check
assert inspect.isclass(result)

# Good — identity check when both paths should resolve to the same object
assert result_a is result_b
```

## Hypothesis Strategies

### ASCII-restrict identifier strategies
Strategies that generate Python identifiers **must** use `max_codepoint=127`. Unicode letters
(`Ā`, `ñ`, etc.) are valid Python identifiers but break on Windows when used as filenames or
embedded in file content (Windows default encoding is cp1252, not UTF-8).

```python
# Correct
python_identifiers = st.text(
    alphabet=st.characters(
        whitelist_categories=("Lu", "Ll"),
        whitelist_characters="_",
        max_codepoint=127,  # ASCII only
    ),
    ...
)
```

### Don't embed generated values inside file content
Use the generated value only at the boundary (e.g. the filename). Keep the file's content static
and controlled.

```python
# Bad — mod_name embedded inside the Python source string
module_content = f'self.value = "{mod_name}"'
module_file.write_text(module_content)

# Good — generated value used only as filename; content is a literal
module_content = "class TestClass:\n    pass\n"
module_file = tmp_path / f"{mod_name}.py"
module_file.write_text(module_content, encoding="utf-8")
```

## File I/O

### Always specify `encoding="utf-8"`
Windows defaults to the system locale encoding (often cp1252). Always pass `encoding="utf-8"` to
`write_text()` and text-mode `open()` calls in tests.

```python
# Correct
path.write_text(content, encoding="utf-8")

# Wrong — silently breaks on Windows with non-ASCII content
path.write_text(content)
```

### Use `tmp_path`, never `tempfile`
Always use pytest's built-in `tmp_path` fixture for temporary files and directories.
Never import or use the `tempfile` standard library module in tests.

## Path Comparisons

Always compare paths using `.as_posix()` on both sides. `str(Path(...))` returns backslashes on
Windows and forward slashes on Linux/macOS, causing cross-platform failures.

```python
# Correct
assert Path(a).as_posix() == Path(b).as_posix()
assert Path(p).as_posix() in other_path.as_posix()

# Wrong
assert str(path_a) == str(path_b)
```
